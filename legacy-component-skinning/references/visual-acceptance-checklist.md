# 验收清单

## A. 非 UI 边界
- [ ] 未修改 API
- [ ] 未修改请求 / 响应结构
- [ ] 未修改数据流
- [ ] 未修改业务逻辑
- [ ] 未修改权限逻辑
- [ ] 未修改路由
- [ ] 未替换原有具体前端组件

## B. 未借用参考实现
- [ ] 未复制参考站点或设计系统的源码、构建产物或样式产物
- [ ] 未热链参考 CSS / JS / 字体 / 图标 / 运行时资源
- [ ] 未仅为贴近样式而引入新的 UI 组件库或样式运行时依赖

## C. 共享层优先
- [ ] 优先通过 theme / token / shared component overrides 完成迁移
- [ ] 未大量散落一次性页面 CSS
- [ ] 同类组件在不同页面视觉一致
- [ ] 同一类视觉修复若跨 2 个以上页面重复出现，已下沉到共享层而不是散落在页面补丁
- [ ] page-level patch 只处理页面特有结构差异，不承载通用皮肤规则
- [ ] 若仍保留多个相似页面补丁，原因已记录且可辩护

## D. 组件矩阵

### Button
- [ ] 高度接近参考
- [ ] 圆角接近参考
- [ ] 字体层级接近参考
- [ ] hover / active / disabled / loading 一致

### Input / Select
- [ ] default / focus / disabled / error 一致
- [ ] placeholder / icon / clear 区域节奏协调
- [ ] 表单项间距一致
- [ ] 空值或 placeholder-only 的 select trigger 宽度稳定
- [ ] 下拉面板的选项、hover、selected 态可清晰识别
- [ ] 已选值、主选项文案和 placeholder 不会长期只剩 1-2 个字或难以识别的前缀
- [ ] trigger 不会为了显示长值被异常拉长并挤压同排关键控件
- [ ] dropdown / popover 面板可按内容合理增宽，但不会撑爆容器或视口
- [ ] 若存在截断，ellipsis 与补充查看路径可用，且不只依赖 hover tooltip

### Table
- [ ] 表头层级一致
- [ ] 行高和密度一致
- [ ] 分割线和 hover 效果一致
- [ ] 分页风格统一

### Modal / Drawer / Popover
- [ ] 标题区、内容区、footer 视觉层级一致
- [ ] radius / shadow / mask 协调
- [ ] 提示 banner / 说明卡片 / 表单块左右边界一致
- [ ] footer 按钮组对齐和间距统一

### Navigation / Tabs / Breadcrumb
- [ ] 选中态统一
- [ ] hover / active 一致
- [ ] 页面层级清晰

## E. 复合业务场景
- [ ] 列表页工具栏中的搜索框、下拉、日期范围、按钮组已对齐
- [ ] 配置表单中的 label、control、help text、必填星号已对齐
- [ ] 弹窗/抽屉中的说明块、表单块、footer 已在同一布局节奏
- [ ] 筛选栏/工具栏里的 select、segmented、date range 等控件既不因过窄导致内容不可读，也不因追求全文展示而异常拉长
- [ ] 横向选项组 / segmented rail 在长文案和窄容器下没有截断、挤压或异常大空白
- [ ] 组内 chip / 徽标 / 次按钮与主文案基线一致
- [ ] disabled / read-only 场景下控件宽度和层级保持稳定
- [ ] 至少 2-3 个真实业务场景已做回看，而不是只验原子组件

## F. 交互后出现的 Surface
- [ ] 已对所有关键可见状态变化入口做 interaction sweep，而不是只点少数几个显眼按钮
- [ ] 交互后新出现的 modal / drawer / popover / tab panel 已做样式回看
- [ ] 隐藏 surface 中的表单、按钮、帮助文案、空态控件没有被漏检
- [ ] 至少保留关键入口的触发前/触发后证据

## G. 异常态 / 降级态
- [ ] 已对关键异常态做 exception sweep，而不是只验 happy path
- [ ] 系统级 banner / alert / 降级提示条已做样式回看
- [ ] 表单校验失败、权限不足、后端不可用等异常反馈没有被漏检
- [ ] 至少保留关键异常态的正常/异常成对证据

## H. 页面层级
- [ ] 页面背景 / surface 层次合理
- [ ] 留白节奏统一
- [ ] 主次信息层级明显
- [ ] 桌面端显示正常
- [ ] 项目要求支持的其他断点显示正常

## I. 真实观察证据
- [ ] 验收结论基于真实渲染、真实交互结果或真实异常态反馈
- [ ] 至少保留关键页面 / 关键状态的前后截图、录屏或等价观察记录
- [ ] 没有仅凭代码、CSS、DOM 或静态推断就宣称“已通过”
- [ ] 无法真实查看的页面或状态已明确标记为未验证

## J. 剩余差异
- [ ] 无法 1:1 还原的差异已记录
- [ ] 差异原因清晰，且未靠替换组件解决
